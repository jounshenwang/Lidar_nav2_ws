`Lidar_nav2_ws` 是一个基于 ROS 2 Humble 的 3D LiDAR 自主导航工作空间，支持 Gazebo 仿真和 Livox MID-360 实机部署。本文档只说明本项目的环境搭建流程，默认系统已安装 ROS 2 Humble。

## 1. 环境要求

推荐环境如下：

| 项目 | 版本 / 说明 |
| --- | --- |
| 操作系统 | Ubuntu 22.04 |
| ROS 2 | Humble Hawksbill |
| Gazebo | Gazebo Classic 11 / ROS 2 Gazebo 插件 |
| 编译工具 | colcon, CMake, GCC/G++ |
| LiDAR | Livox MID-360，仅实机模式需要 |

编译或运行前先加载 ROS 2 环境：

```bash
source /opt/ros/humble/setup.bash
printenv ROS_DISTRO
```

期望输出：

```text
humble
```

安装基础工具：

```bash
sudo apt update
sudo apt install -y \
  git wget curl vim gnupg lsb-release software-properties-common \
  build-essential cmake ninja-build pkg-config \
  python3-pip python3-colcon-common-extensions python3-rosdep
```

如果 `rosdep` 尚未初始化：

```bash
sudo rosdep init
rosdep update
```

如果当前网络环境下 `rosdep update` 较慢，可以使用 `rosdepc`：

```bash
sudo pip3 install rosdepc
sudo rosdepc init
rosdepc update
```

## 2. 获取代码

克隆工作空间：

```bash
cd ~
git clone https://github.com/Ikunio/Lidar_nav2_ws.git
cd Lidar_nav2_ws
```

如果工作空间已经存在，直接进入目录：

```bash
cd ~/Lidar_nav2_ws
```

期望目录结构：

```text
Lidar_nav2_ws/
├── src/
├── build.sh
├── mapping_sim.sh
├── nav2_sim.sh
├── mapping_real.sh
└── nav2_real.sh
```

## 3. 安装依赖

优先使用 `rosdep` 根据 `package.xml` 安装依赖：

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
rosdep install --from-paths src --ignore-src --rosdistro humble -r -y
```

也可以使用 `rosdepc`：

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
rosdepc install --from-paths src --ignore-src --rosdistro humble -r -y
```

`rosdep` 会覆盖 `package.xml` 中声明的大部分 ROS 2、PCL、OpenCV、GTSAM、OpenMP、glog 和消息依赖。如果编译时仍提示缺少系统库，再按报错补装；常见补装项如下：

```bash
sudo apt install -y \
  libtbb-dev \
  libboost-all-dev \
  qtbase5-dev qtbase5-private-dev \
  python3-tk
```

如果缺少 Gazebo 或导航相关包：

```bash
sudo apt install -y \
  ros-humble-gazebo-ros-pkgs \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-slam-toolbox \
  ros-humble-pointcloud-to-laserscan \
  ros-humble-tf2-tools
```

## 4. 可选依赖

### 4.1 Livox-SDK2

`src/livox_ros_driver2` 已内置预编译 Livox-SDK2，通常不需要额外安装 SDK。

实机部署前检查 MID-360 网络配置：

```bash
vim src/livox_ros_driver2/config/MID360_config.json
```

重点确认：

- `host_net_info` 中的主机 IP 与 LiDAR 网卡一致。
- `lidar_configs` 中的 LiDAR IP 与实际 MID-360 一致。
- 主机和 LiDAR 位于同一网段。

### 4.2 KISS-Matcher

`nav2_sim.sh` 和 `nav2_real.sh` 默认使用 `global_relocalization_kiss_matcher`。如果编译时报缺少 `kiss_matcher` 或 `robin`，安装 C++ 库：

```bash
cd ~/Lidar_nav2_ws/src/registration/KISS-Matcher
make deps
make cppinstall
```

如果 ROBIN 已安装且 `make cppinstall` 失败：

```bash
make cppinstall_matcher_only
```

### 4.3 small_gicp

如果编译时报缺少 `small_gicp`，从源码安装：

```bash
cd /tmp
git clone https://github.com/koide3/small_gicp.git
cd small_gicp
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(nproc)
sudo cmake --install build
sudo ldconfig
```

## 5. 编译

编译工作空间：

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
./build.sh
```

`build.sh` 等价于：

```bash
colcon build --symlink-install --cmake-args -DCMAKE_BUILD_TYPE=Release
```

编译完成后加载工作空间：

```bash
source install/setup.bash
```

可选：写入 `~/.bashrc`，让新终端自动加载工作空间：

```bash
echo "source ~/Lidar_nav2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

## 6. 快速检查

检查关键包是否可被 ROS 2 发现：

```bash
ros2 pkg list | grep -E "me_nav2_bringup|lio_interface|fast_lio|livox_ros_driver2|global_relocalization_kiss_matcher"
```

检查启动文件参数：

```bash
ros2 launch me_nav2_bringup my_nav2_launch.py --show-args
ros2 launch lio_interface lio_interface_launch.py --show-args
```

检查 TF 工具：

```bash
ros2 run tf2_tools view_frames
```

## 7. 运行

### 7.1 仿真建图

启动仿真建图：

```bash
cd ~/Lidar_nav2_ws
source install/setup.bash
./mapping_sim.sh
```

该脚本会启动 Gazebo、GUI 遥控、FAST-LIO、`lio_interface`、`sensor_scan_generation`、`pointcloud_to_laserscan`、`slam_toolbox` 和 Nav2。

运行状态检查：

```bash
ros2 topic hz /scan
ros2 topic hz /registered_scan
ros2 run tf2_ros tf2_echo odom base_footprint
```

驾驶机器人遍历环境后保存地图：

```bash
./save_map.sh
./save_pcd.sh
```

地图输出位置：

```text
2D map: src/me_nav2_bringup/map/
3D PCD: src/me_nav2_bringup/pcd/
```

### 7.2 仿真导航

导航前检查地图路径：

```bash
vim src/me_nav2_bringup/launch/my_nav2_launch.py
vim src/registration/global_relocalization_kiss_matcher/launch/global_kiss_matcher_relocalization_launch.py
vim src/registration/small_gicp_relocalization/launch/small_gicp_relocalization_launch.py
```

重点确认：

- `map_yaml_file` 指向要使用的 `.yaml` 地图。
- `prior_pcd_file` 指向要使用的 `.pcd` 点云地图。
- 同一时间只允许一个节点发布 `map -> odom`。

启动仿真导航：

```bash
cd ~/Lidar_nav2_ws
source install/setup.bash
./nav2_sim.sh
```

运行状态检查：

```bash
ros2 run tf2_ros tf2_echo map odom
ros2 topic hz /registered_scan
```

在 RViz 中使用 `Nav2 Goal` 发送导航目标。

### 7.3 实机运行

实机建图或导航前确认：

- MID-360 网络连通：`ping <MID360_IP>`。
- `src/livox_ros_driver2/config/MID360_config.json` 配置正确。
- 机器人电源、LiDAR、IMU、底盘控制器、急停和人工接管正常。
- 已加载工作空间：`source ~/Lidar_nav2_ws/install/setup.bash`。

实机建图：

```bash
cd ~/Lidar_nav2_ws
./mapping_real.sh
```

实机导航：

```bash
cd ~/Lidar_nav2_ws
./nav2_real.sh
```

## 8. LIO 后端切换

当前脚本默认使用 FAST-LIO。如需切换 Point-LIO，编辑以下脚本中的启动块：

```bash
vim mapping_sim.sh
vim nav2_sim.sh
vim mapping_real.sh
vim nav2_real.sh
```

后端差异：

| 项目 | FAST-LIO | Point-LIO |
| --- | --- | --- |
| 仿真 LiDAR 输入 | 直接使用 PointCloud2 | 需要 `ign_sim_pointcloud_tool` |
| 实机 LiDAR 驱动 | `fast_lio_msg_MID360_launch.py` | `point_lio_msg_MID360_launch.py` |
| 接口启动文件 | `fastlio_lio_interface_launch.py` | `pointlio_lio_interface_launch.py` |
| Point-LIO 配置 | 不使用 | `mid360_sim.yaml` / `mid360_real.yaml` |

不要混用 FAST-LIO 和 Point-LIO 的 `lidar_type` 配置。两者的枚举定义不同。

## 9. 常见问题

### 找不到 ROS 2 包

```text
Package 'xxx' not found
```

处理：

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 pkg list | grep xxx
```

如果仍找不到：

```bash
./build.sh
source install/setup.bash
```

### Gazebo 残留进程

```bash
killall -9 gzserver gzclient
./mapping_sim.sh
```

### 缺少 `kiss_matcher` 或 `robin`

```bash
cd ~/Lidar_nav2_ws/src/registration/KISS-Matcher
make deps
make cppinstall
cd ~/Lidar_nav2_ws
./build.sh
```

### 缺少 `small_gicp`

```bash
cd /tmp
git clone https://github.com/koide3/small_gicp.git
cd small_gicp
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(nproc)
sudo cmake --install build
sudo ldconfig

cd ~/Lidar_nav2_ws
./build.sh
```

### `/scan` 或 `/registered_scan` 无数据

按顺序检查话题：

```bash
ros2 topic hz /livox/lidar
ros2 topic hz /livox/lidar/pointcloud
ros2 topic hz /registered_scan
ros2 topic hz /scan
```

常见原因：

- LiDAR 驱动未启动，或网络配置错误。
- LIO 节点没有发布里程计。
- `lio_interface` 订阅了错误的 LIO 话题。
- `sensor_scan_generation` 没有收到点云或 TF。
- `pointcloud_to_laserscan` 高度切片参数不合适。

### Nav2 无法规划或机器人不动

检查 TF 和速度命令：

```bash
ros2 run tf2_ros tf2_echo map odom
ros2 run tf2_ros tf2_echo odom base_footprint
ros2 topic hz /scan
ros2 topic echo /cmd_vel
```

常见原因：

- 缺少 `map -> odom`，重定位尚未成功。
- 缺少 `odom -> base_footprint`，LIO 或桥接异常。
- 2D 地图和 PCD 点云不是同一次建图结果。
- RViz 初始位姿或目标点不在可通行区域。
- 底盘没有消费 `/cmd_vel`，或安全状态未解除。
